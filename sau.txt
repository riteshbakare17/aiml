Q.1 Find the Shortest Path in an Unweighted Graph from Source to Destination using BFS
________________________________________
✅ 1. Python Algorithm
Python
def bfs_shortest_path(graph, start, end):
    if start == end:
        return [start]
    queue = [[start]]
    visited = set([start])
    while queue:
        path = queue.pop(0)
        node = path[-1]
        for neighbor in graph.get(node, []):
            if neighbor not in visited:
                new_path = path + [neighbor]
                if neighbor == end:
                    return new_path
                queue.append(new_path)
                visited.add(neighbor)
    return None

# ➤ USER INPUT
print("=== BFS Shortest Path ===")
n = int(input("Enter number of edges: "))
graph = {}
for _ in range(n):
    u, v = map(int, input("Enter edge (u v): ").split())
    graph.setdefault(u, []).append(v)
    graph.setdefault(v, []).append(u)  # undirected

start = int(input("Enter start node: "))
end = int(input("Enter end node: "))

# ➤ OUTPUT
path = bfs_shortest_path(graph, start, end)
if path:
    print(f"Shortest path from {start} to {end}: {' → '.join(map(str, path))}")
else:
    print("No path found.")________________________________________

❓ 4. Important Viva Questions
Q1: Why use BFS for shortest path in unweighted graphs?
A: BFS explores all nodes level by level (by distance from source). So when it first reaches the destination, it must be via the shortest path (minimum edges). In unweighted graphs, shortest path = least number of edges.
________________________________________
Q2: What’s the time and space complexity?
A:
•	Time: O(V + E) — we visit each vertex and edge at most once.
•	Space: O(V) — for queue, visited set, and parent map.
________________________________________
Q3: Can DFS be used to find shortest path?
A: No, DFS goes deep first and may find a longer path before a shorter one. It doesn't guarantee minimality in path length for unweighted graphs.
________________________________________
Q4: What if the graph is disconnected?
A: BFS will only explore the connected component of the source. If destination is in another component, it returns None or -1, correctly indicating no path exists.
________________________________________
Q5: How would you modify this for a weighted graph?
A: For weighted graphs (with non-negative weights), use Dijkstra’s Algorithm with a priority queue. BFS doesn’t work because “level” doesn’t correspond to total weight anymore.
________________________________________
Q6: Why do we need the parent dictionary?
A: To reconstruct the actual path after reaching the destination. Without it, we only know the distance, not the route taken.
________________________________________
Q7: What happens if start == end?
A: We immediately return [start] — the path of length 0. This is an important edge case to handle.
________________________________________
Q8: Can there be multiple shortest paths? Does your code return all?
A: Yes, multiple shortest paths can exist (same number of edges). This code returns only one — the one found first by BFS traversal order. To find all, you’d need to continue BFS until finishing the current level and collect all paths of same minimal length.

Q.2 Implement DFS to Explore All Reachable Nodes and Detect Cycles
________________________________________
✅ 1. Python Algorithm
Python
def dfs_cycle_detect(graph, start):
    visited = set()
    rec_stack = set()
    
    def dfs(node):
        visited.add(node)
        rec_stack.add(node)
        for neighbor in graph.get(node, []):
            if neighbor not in visited:
                if dfs(neighbor):
                    return True
            elif neighbor in rec_stack:
                return True
        rec_stack.remove(node)
        return False
    
    return dfs(start)

# ➤ USER INPUT
print("=== DFS Cycle Detection ===")
n = int(input("Enter number of edges: "))
graph = {}
for _ in range(n):
    u, v = map(int, input("Enter directed edge (u → v): ").split())
    graph.setdefault(u, []).append(v)

start = int(input("Enter start node: "))

# ➤ OUTPUT
has_cycle = dfs_cycle_detect(graph, start)
print("Cycle detected:" , has_cycle)

❓ 4. Important Viva Questions
Q1: Why does DFS detect cycles using a recursion stack?
A: The recursion stack holds nodes currently in the DFS traversal path. If we encounter a node already in this stack, it means we’ve found a back edge, which creates a cycle. This works for both directed and undirected graphs.
________________________________________
Q2: What’s the difference between “visited” and “recursion stack”?
A:
•	visited: Permanent marker — node has been fully processed.
•	rec_stack: Temporary — node is in the current DFS path. Removed after returning from recursion.
A cycle is detected only when a neighbor is in rec_stack, not just visited.
________________________________________
Q3: Can DFS be used for shortest path in unweighted graphs?
A: No. DFS explores depth-first, so it may find a longer path first. Use BFS for shortest path in unweighted graphs.
________________________________________
Q4: Time and space complexity?
A:
•	Time: O(V + E) — each vertex and edge processed once.
•	Space: O(V) — for visited, recursion stack, and system call stack (depth up to V).
________________________________________
Q5: How to modify this for undirected graphs?
A: Already handled! In undirected graphs, we must avoid considering parent as a back edge. But in our code, since we don’t pass parent info, it still works because:
•	We only check if neighbor is in rec_stack.
•	Parent won’t be in rec_stack when processing child (it was popped already? Actually — careful!)
⚠️ Correction: In undirected graphs, without tracking parent, you’ll falsely detect cycle because (u,v) and (v,u) exist.
Better approach for undirected:
Python
def dfs(node, parent):
    visited.add(node)
    for neighbor in self.graph[node]:
        if neighbor == parent: continue
        if neighbor in visited: return True  # cycle
        if dfs(neighbor, node): return True
    return False
________________________________________
Q6: What if the graph is disconnected?
A: DFS from one node only explores its connected component. To explore entire graph, iterate over all unvisited nodes and start DFS from each — as shown in find_all_components_with_cycles().
________________________________________
Q7: Can DFS detect all cycles?
A: It detects if a cycle exists in the component. To list all cycles, you’d need to store paths or use more advanced algorithms like Johnson’s algorithm.
________________________________________
Q8: What’s the role of ‘reachable’ set?
A: It collects all nodes accessible from the start node. Useful for connectivity analysis, component identification, and reachability queries.

Q.3 Find Least-Cost Path in Weighted Graph using Uniform Cost Search
⚠️ Note: Uniform Cost Search is essentially Dijkstra’s Algorithm without a goal check during expansion — it expands the least-cost node first until reaching the target.
________________________________________
✅ 1. Python Algorithm
Python
def ucs(graph, start, goal):
    frontier = [(0, start, [start])]
    while frontier:
        frontier.sort()
        cost, node, path = frontier.pop(0)
        if node == goal:
            return cost, path
        for neighbor, edge_cost in graph.get(node, []):
            new_cost = cost + edge_cost
            new_path = path + [neighbor]
            frontier.append((new_cost, neighbor, new_path))
    return -1, []

# ➤ USER INPUT
print("=== Uniform Cost Search ===")
n = int(input("Enter number of weighted edges: "))
graph = {}
for _ in range(n):
    u, v, w = input("Enter edge (u v weight): ").split()
    w = int(w)
    graph.setdefault(u, []).append((v, w))
    graph.setdefault(v, []).append((u, w))  # undirected

start = input("Enter start node: ")
goal = input("Enter goal node: ")

# ➤ OUTPUT
cost, path = ucs(graph, start, goal)
if cost != -1:
    print(f"Least cost: {cost}")
    print(f"Path: {' → '.join(path)}")
else:
    print("No path found.")

❓ 4. Important Viva Questions
Q1: What is Uniform Cost Search?
A: UCS is a search algorithm that expands the least-cost node first. It guarantees finding the lowest-cost path in a weighted graph with non-negative edge weights. It’s identical to Dijkstra’s algorithm when run to completion.
________________________________________
Q2: Why use a priority queue?
A: To always expand the node with the smallest cumulative cost so far. This greedy choice ensures optimality — the first time we reach the goal, it’s via the least-cost path.
________________________________________
Q3: Can UCS handle negative edge weights?
A: ❌ No. Like Dijkstra, UCS assumes non-negative weights. With negative weights, a later path might be cheaper, breaking the “first visit = optimal” assumption. Use Bellman-Ford instead.
________________________________________
Q4: Time and space complexity?
A:
•	Time: O((V + E) log V) — each node/edge may be pushed into heap, heap op is log V.
•	Space: O(V) — for visited set, and up to O(E) elements in heap.
________________________________________
Q5: Difference between UCS and BFS?
A:
•	BFS: Uses a simple queue, finds shortest path in unweighted graphs (fewest edges).
•	UCS: Uses priority queue, finds least-cost path in weighted graphs.
•	If all weights = 1, UCS behaves like BFS.
________________________________________
Q6: Why do we check if node in visited: continue?
A: Once a node is popped from the priority queue, we’ve found its minimal cost. Later entries for the same node (with higher cost) can be safely ignored. This optimization avoids redundant processing.
________________________________________
Q7: What if multiple paths have same cost?
A: UCS will return one of them (the one expanded first). To get all minimal-cost paths, you’d need to modify the algorithm to continue exploring at the same cost level.
________________________________________
Q8: Is UCS complete and optimal?
A:
•	✅ Complete: Yes, if solution exists and edge costs ≥ ε > 0 (avoids infinite descent).
•	✅ Optimal: Yes, always finds least-cost path (with non-negative weights).

Q.4 Search for Target Node in Tree using Depth Limited Search (DLS)
💡 Depth Limited Search is DFS with a depth bound — it won’t explore beyond a specified depth. Useful to avoid infinite paths and control resource usage.
________________________________________
✅ 1. Python Algorithm
Python
def dls(tree, target, depth_limit, node, path=None, depth=0):
    if path is None: path = [node]
    if node == target: return True, path
    if depth >= depth_limit: return False, []
    for child in tree.get(node, []):
        found, result_path = dls(tree, target, depth_limit, child, path + [child], depth + 1)
        if found: return True, result_path
    return False, []

# ➤ USER INPUT
print("=== Depth Limited Search ===")
n = int(input("Enter number of parent-child relationships: "))
tree = {}
for _ in range(n):
    line = input("Enter 'parent child1 child2 ...': ").split()
    parent = line[0]
    children = line[1:]
    tree[parent] = children

start = input("Enter root node: ")
target = input("Enter target node: ")
max_depth = int(input("Enter depth limit: "))

# ➤ OUTPUT
found, path = dls(tree, target, max_depth, start)
if found:
    print(f"Found '{target}' at depth {len(path)-1}: {' → '.join(path)}")
else:
    print(f"'{target}' not found within depth {max_depth}.")

❓ 4. Important Viva Questions
Q1: What is Depth Limited Search?
A: DLS is a DFS variant that imposes a maximum depth limit. It does not explore nodes beyond this limit, preventing infinite descent in cyclic graphs or overly deep trees.
________________________________________
Q2: Why use DLS instead of regular DFS?
A:
•	To avoid infinite loops in graphs with cycles.
•	To control memory/time usage.
•	Foundation for Iterative Deepening Search (IDS) — run DLS with increasing depth limits.
________________________________________
Q3: Is DLS complete? Optimal?
A:
•	✅ Complete if solution depth ≤ limit.
•	❌ Not complete if solution is deeper than limit.
•	❌ Not optimal — may find a deeper goal before a shallower one if traversal order differs (though in trees without cost, “optimal” usually means shallowest).
________________________________________
Q4: Time and space complexity?
A:
•	Time: O(b^d) where b = branching factor, d = depth limit.
•	Space: O(d) — only store current path in recursion stack.
________________________________________
Q5: What happens if depth limit is too small?
A: DLS will fail to find the target even if it exists — returns “not found”. This is called a cutoff.
________________________________________
Q6: Difference between DLS and DFS?
A: DFS has no depth bound — can go infinitely deep. DLS stops at a predefined depth. In finite trees without cycles, they behave similarly if limit ≥ tree depth.
________________________________________
Q7: Can DLS be used in graphs with cycles?
A: Yes! Unlike DFS, DLS won’t loop infinitely because of the depth bound. But you may still want to track visited nodes per path to avoid redundant cycles within the depth limit.
________________________________________
Q8: What is depth of the root node?
A: Depth = 0 (standard convention). Some systems use depth=1 — always clarify! Our implementation uses 0.
Q.5 : Implement Iterative Deepening DFS to Find Goal at Minimum Depth
💡 IDDFS runs Depth-Limited Search repeatedly with increasing depth limits (0, 1, 2, ...) until the goal is found. Guarantees finding the shallowest goal like BFS, but uses memory like DFS.
________________________________________
✅ 1. Python Algorithm
Python
# Reuse dls from above

def iddfs(tree, target, start, max_depth=10):
    for depth in range(max_depth + 1):
        found, path = dls(tree, target, depth, start)
        if found:
            return True, path, depth
    return False, [], -1

# ➤ USER INPUT (same tree input as DLS)

for _ in range(n):

    parent = line[0]
    children = line[1:]
    tree[parent] = children

target = input("Enter target node: ")
max_depth = int(input("Enter max search depth: "))

# ➤ OUTPUT

if found:
    print(f"Found '{target}' at minimum depth {depth}: {' → '.join(path)}")
else:
    print(f"'{target}' not found within depth {max_depth}.")

❓ 4. Important Viva Questions
Q1: What is Iterative Deepening DFS?
A: IDDFS combines DFS’s space efficiency with BFS’s completeness and shallowness guarantee. It runs DLS iteratively with depth limits 0, 1, 2,... until goal is found.
________________________________________
Q2: Why is IDDFS better than BFS in large trees?
A: BFS uses O(b^d) memory to store all nodes at level d. IDDFS uses only O(d) memory (like DFS) since it doesn’t store all siblings — just the current path.
________________________________________
Q3: Doesn’t repeating search at each level waste time?
A: Surprisingly, no — in a branching tree, most work is done at the deepest level. Total cost ≈ O(b^d), same as BFS. The shallower levels contribute negligibly:
b^0 + b^1 + ... + b^d = (b^(d+1)-1)/(b-1) ≈ b^d for large d.
________________________________________
Q4: Is IDDFS complete and optimal?
A:
•	✅ Complete: Yes, if goal exists at finite depth.
•	✅ Optimal: For unit edge costs (or unweighted trees), yes — finds shallowest goal.
•	❌ Not optimal for weighted graphs — use UCS/Dijkstra instead.
________________________________________
Q5: Time and space complexity?
A:
•	Time: O(b^d) — where b = branching factor, d = depth of shallowest goal.
•	Space: O(d) — maximum depth of recursion stack.
________________________________________
Q6: When should you use IDDFS?
A: When:
•	Search space is large or infinite.
•	Memory is limited.
•	You want the shallowest solution (e.g., fewest moves in puzzles).
•	Commonly used in game AI (Chess, Go) and puzzle solvers (Rubik’s cube).
________________________________________
Q7: Can IDDFS handle graphs with cycles?
A: Yes, but you should track visited nodes per iteration or per path to avoid looping. In trees (no cycles), it’s safe without visited tracking.
________________________________________
Q8: What’s the main disadvantage?
A: Repeated exploration of shallow nodes. But as explained, this overhead is asymptotically negligible compared to the deepest level.

Q.6 Implement A* Search to Find Optimal Path Using Heuristic
💡 A* combines Uniform Cost Search (actual cost g(n)) with Heuristic Guidance (h(n)) → total cost f(n) = g(n) + h(n).
Guarantees optimality if heuristic is admissible (never overestimates true cost).
________________________________________
✅ 1. Python Algorithm
Python
def astar(graph, start, goal, positions):
    def manhattan(n1, n2):
        x1, y1 = positions[n1]
        x2, y2 = positions[n2]
        return abs(x1 - x2) + abs(y1 - y2)
    
    open_list = [(manhattan(start, goal), 0, start, [start])]
    while open_list:
        open_list.sort()
        f, g, node, path = open_list.pop(0)
        if node == goal:
            return g, path
        for neighbor, cost in graph.get(node, []):
            new_g = g + cost
            new_f = new_g + manhattan(neighbor, goal)
            open_list.append((new_f, new_g, neighbor, path + [neighbor]))
    return -1, []

# ➤ USER INPUT
print("=== A* Search ===")

❓ 4. Important Viva Questions
Q1: What is A* algorithm?
A: A* is an informed search algorithm that finds the least-cost path from start to goal by combining:
•	g(n): actual cost from start to node n
•	h(n): estimated cost from n to goal
•	Uses f(n) = g(n) + h(n) to prioritize nodes.
________________________________________
Q2: What does “admissible heuristic” mean?
A: A heuristic is admissible if it never overestimates the true cost to reach the goal. This guarantees A* finds the optimal path. Euclidean and Manhattan distances are admissible for grid/graph pathfinding.
________________________________________
Q3: What is a consistent (monotonic) heuristic?
A: A heuristic is consistent if for every node n and successor n' with step cost c:
h(n) ≤ c + h(n')
Consistency implies admissibility and ensures that once a node is expanded, its cost is optimal (no need to revisit).
________________________________________
Q4: Time and space complexity?
A:
•	Time: Exponential in worst case O(b^d), but good heuristics drastically reduce it.
•	Space: O(b^d) — stores all generated nodes in memory (main drawback).
________________________________________
Q5: Difference between A* and UCS?
A: UCS uses only g(n) (like A* with h(n)=0). A* uses f(n)=g(n)+h(n), so it’s guided toward goal — usually expands fewer nodes.
________________________________________
Q6: What if heuristic overestimates?
A: A* may return a suboptimal path. Optimality guaranteed only with admissible heuristics.
________________________________________
Q7: Can A* be used in non-graph environments?
A: Yes! A* is widely used in:
•	Game AI (pathfinding in maps)
•	Robotics (motion planning)
•	Puzzle solving (15-puzzle, Rubik’s cube)
•	Network routing
________________________________________
Q8: What happens if no path exists?
A: A* will exhaust all reachable nodes and return failure — same behavior as BFS/DFS.

Q.7 Solve N-Queens using Hill Climbing
💡 N-Queens: Place N queens on an N×N chessboard so that no two queens attack each other.
Hill Climbing: Start with a random configuration, then iteratively move one queen to reduce the number of attacking pairs (heuristic = minimize conflicts).

import random

def count_conflicts(board):
    n = len(board)
    conflicts = 0
    for i in range(n):
        for j in range(i+1, n):
            if board[i] == board[j] or abs(board[i]-board[j]) == abs(i-j):
                conflicts += 1
    return conflicts

def solve_nqueens(n, max_iter=1000):
    board = [random.randint(0, n-1) for _ in range(n)]
    for _ in range(max_iter):
        if count_conflicts(board) == 0:
            return board
        best_board = board[:]
        best_conflicts = count_conflicts(board)
        for col in range(n):
            for row in range(n):
                if row == board[col]: continue
                new_board = board[:]
                new_board[col] = row
                conf = count_conflicts(new_board)
                if conf < best_conflicts:
                    best_conflicts = conf
                    best_board = new_board
        if best_board == board: break
        board = best_board
    return board

def check_winner(board):
    for i in range(3):
        if board[i][0] == board[i][1] == board[i][2] != ' ': return board[i][0]
        if board[0][i] == board[1][i] == board[2][i] != ' ': return board[0][i]
    if board[0][0] == board[1][1] == board[2][2] != ' ': return board[0][0]
    if board[0][2] == board[1][1] == board[2][0] != ' ': return board[0][2]

def minimax(board, depth, is_maximizing):
    result = check_winner(board)
    if result == 'O': return 10 - depth
    if result == 'X': return depth - 10
    if result == 'Tie': return 0
    if is_maximizing:
        best = -float('inf')
        for i in range(3):
            for j in range(3):
                if board[i][j] == ' ':
                    board[i][j] = 'O'
                    score = minimax(board, depth+1, False)
                    board[i][j] = ' '
                    best = max(best, score)

    else:
        best = float('inf')
        for i in range(3):
            for j in range(3):
                if board[i][j] == ' ':
                    board[i][j] = 'X'
                    score = minimax(board, depth+1, True)
                    board[i][j] = ' '
                    best = min(best, score)
        return best

📝 How Alpha-Beta Pruning Works (On Paper)
Let’s say we’re evaluating this position (AI to move):
text
X | O | X
---------
O | O | 
---------
X |   | 
AI ('O') can play at (1,2), (2,1), or (2,2).
Without pruning:
Minimax evaluates ALL possible future moves recursively.
With Alpha-Beta:
•	Start evaluating move (1,2) 
o	Suppose it leads to AI win → score = +9
o	Set α = 9 (best so far for Max)
•	Now evaluate move (2,1) 
o	As we explore, suppose we find a path where human can force a win → score ≤ -8
o	Since β (-8) ≤ α (9) → we prune (stop evaluating) this branch!
•	Skip unnecessary computation.
💡 Result: Same optimal move chosen, but much faster!

def solve_sudoku(board):
    # Find empty cell
    for r in range(9):
        for c in range(9):
            if board[r][c] == 0:
                # Try digits 1-9
                for num in range(1, 10):
                    if is_valid(board, r, c, num):
                        board[r][c] = num
                        if solve_sudoku(board):
                            return True
                        board[r][c] = 0  # backtrack
                return False  # trigger backtracking
    return True  # all filled = solved

def is_valid(board, row, col, num):
    # Check row
    for x in range(9):
        if board[row][x] == num:
            return False
    # Check column
    for x in range(9):
        if board[x][col] == num:
            return False
    # Check 3x3 box
    start_row, start_col = 3 * (row // 3), 3 * (col // 3)
    for i in range(3):
        for j in range(3):
            if board[start_row + i][start_col + j] == num:
                return False
    return True

from collections import deque

def find_zero(state):
    for i in range(3):
        for j in range(3):
            if state[i][j] == 0:
                return i, j

def get_neighbors(state):
    neighbors = []
    x, y = find_zero(state)
    moves = [(-1,0), (1,0), (0,-1), (0,1)]  # up, down, left, right
    
    for dx, dy in moves:
        nx, ny = x + dx, y + dy
        if 0 <= nx < 3 and 0 <= ny < 3:
            # Create new state by swapping
            new_state = [row[:] for row in state]  # deep copy
            new_state[x][y], new_state[nx][ny] = new_state[nx][ny], new_state[x][y]
            neighbors.append(new_state)
    return neighbors

def state_to_tuple(state):
    return tuple(tuple(row) for row in state)

def solve_8puzzle(start, goal):
    queue = deque([(start, [])])  # (state, path)
    visited = set()
    visited.add(state_to_tuple(start))
    
    while queue:
        state, path = queue.popleft()
        
        if state == goal:
            return path
        
        for neighbor in get_neighbors(state):
            n_tuple = state_to_tuple(neighbor)
            if n_tuple not in visited:
                visited.add(n_tuple)
                queue.append((neighbor, path + [neighbor]))
    
    return None  # no solution

from collections import deque

def print_maze(maze, path=None):
    if path is None: path = set()
    for i, row in enumerate(maze):
        line = ""
        for j, cell in enumerate(row):
            if (i, j) in path:
                line += "* "  # path
            else:
                line += "█ " if cell == 1 else ". "  # wall or empty
        print(line)

def get_neighbors(maze, r, c):
    neighbors = []
    for dr, dc in [(-1,0), (1,0), (0,-1), (0,1)]:
        nr, nc = r + dr, c + dc
        if 0 <= nr < len(maze) and 0 <= nc < len(maze[0]) and maze[nr][nc] == 0:
            neighbors.append((nr, nc))
    return neighbors

# ===== BFS =====
def bfs(maze, start, goal):
    queue = deque([(start, [start])])
    visited = set([start])
    
    while queue:
        (r, c), path = queue.popleft()
        if (r, c) == goal:
            return path
        
        for nr, nc in get_neighbors(maze, r, c):
            if (nr, nc) not in visited:
                visited.add((nr, nc))
                queue.append(((nr, nc), path + [(nr, nc)]))
    return None

# ===== DFS =====
def dfs(maze, start, goal):
    stack = [(start, [start])]
    visited = set()
    
    while stack:
        (r, c), path = stack.pop()
        if (r, c) == goal:
            return path
        
        if (r, c) in visited:
            continue
        visited.add((r, c))
        
        for nr, nc in get_neighbors(maze, r, c):
            if (nr, nc) not in visited:
                stack.append(((nr, nc), path + [(nr, nc)]))
    return None

# ===== UCS (Uniform Cost Search) =====
# For maze, all steps cost 1 → same as BFS, but shown for structure
def ucs(maze, start, goal):
    # (cost, position, path)
    frontier = [(0, start, [start])]
    visited = set()
    
    while frontier:
        frontier.sort()  # sort by cost (simulate priority queue)
        cost, (r, c), path = frontier.pop(0)
        
        if (r, c) == goal:
            return path
        
        if (r, c) in visited:
            continue
        visited.add((r, c))
        
        for nr, nc in get_neighbors(maze, r, c):
            if (nr, nc) not in visited:
                frontier.append((cost + 1, (nr, nc), path + [(nr, nc)]))
    return None

def is_safe(board, row, col, n):
    # Check column
    for i in range(row):
        if board[i][col] == 1:
            return False
    
    # Check upper-left diagonal
    for i, j in zip(range(row, -1, -1), range(col, -1, -1)):
        if board[i][j] == 1:
            return False
    
    # Check upper-right diagonal
    for i, j in zip(range(row, -1, -1), range(col, n)):
        if board[i][j] == 1:
            return False
    
    return True

def solve_nqueens_util(board, row, n):
    if row >= n:
        return True  # All queens placed
    
    for col in range(n):
        if is_safe(board, row, col, n):
            board[row][col] = 1  # Place queen
            
            if solve_nqueens_util(board, row + 1, n):
                return True
            
            board[row][col] = 0  # Backtrack
    
    return False

def solve_nqueens(n):
    board = [[0] * n for _ in range(n)]
    
    if not solve_nqueens_util(board, 0, n):
        print("No solution exists.")
        return None
    
    return board

    if board is None: return
    n = len(board)
    for i in range(n):
        line = ""
        for j in range(n):
            line += "Q " if board[i][j] == 1 else ". "
        print(line)

import random
import math

def distance(city1, city2):
    return math.sqrt((city1[0]-city2[0])**2 + (city1[1]-city2[1])**2)

def total_distance(tour, cities):
    d = 0
    for i in range(len(tour)):
        d += distance(cities[tour[i]], cities[tour[(i+1) % len(tour)]])
    return d

def random_tour(n):
    tour = list(range(n))
    random.shuffle(tour)
    return tour

def get_neighbors(tour):
    neighbors = []
    for i in range(len(tour)):
        for j in range(i + 1, len(tour)):
            neighbor = tour[:]
            neighbor[i], neighbor[j] = neighbor[j], neighbor[i]  # swap
            neighbors.append(neighbor)
    return neighbors

def hill_climb_tsp(cities, max_iter=1000):
    n = len(cities)
    current_tour = random_tour(n)
    current_dist = total_distance(current_tour, cities)
    
    for _ in range(max_iter):
        neighbors = get_neighbors(current_tour)
        best_neighbor = None
        best_dist = current_dist
        
        for neighbor in neighbors:
            dist = total_distance(neighbor, cities)
            if dist < best_dist:
                best_dist = dist
                best_neighbor = neighbor
        
        if best_neighbor is None:
            break  # local optimum
        current_tour = best_neighbor
        current_dist = best_dist
    
    return current_tour, current_dist

# ➤ SAMPLE INPUT

    (0, 0),   # A
    (1, 3),   # B
    (4, 3),   # C
    (6, 0),   # D

]

print("Cities (x,y):")
for i, (x, y) in enumerate(cities):
    print(f"  City {i}: ({x}, {y})")

tour, length = hill_climb_tsp(cities)

print(f"📏 Total distance: {length:.2f}")

Q.15 Write a program to implement a water jug problem.
from collections import deque

def solve_water_jug(jug1_cap, jug2_cap, target):
    # BFS from (0, 0)
    queue = deque([((0, 0), [])])  # (state, path)
    visited = set()
    visited.add((0, 0))
    
    while queue:
        (a, b), path = queue.popleft()
        
        # Check if target reached in either jug
        if a == target or b == target:
            return path + [(a, b)]
        
        # Generate all possible next states
        next_states = [
            (jug1_cap, b),    # Fill jug1
            (a, jug2_cap),    # Fill jug2
            (0, b),           # Empty jug1
            (a, 0),           # Empty jug2
            # Pour jug1 → jug2
            (max(0, a - (jug2_cap - b)), min(jug2_cap, b + a)),
            # Pour jug2 → jug1
            (min(jug1_cap, a + b), max(0, b - (jug1_cap - a)))
        ]
        
        for state in next_states:
            if state not in visited:
                visited.add(state)
                queue.append((state, path + [(a, b)]))
    
    return None  # No solution

# ➤ SAMPLE INPUT
jug1 = 4
jug2 = 3
target = 2

print(f"Jug1 capacity: {jug1}L")
print(f"Jug2 capacity: {jug2}L")
print(f"Target: {target}L in either jug\n")

solution = solve_water_jug(jug1, jug2, target)

if solution:
    print("✅ Solution found:")
    for i, (a, b) in enumerate(solution):
        print(f"Step {i}: Jug1={a}L, Jug2={b}L")
else:
    print("❌ No solution exists.")

❓ Important Viva Questions
Q1: What is the Water Jug Problem?
A: Given two jugs of known capacities, find a sequence of fill/empty/pour operations to measure an exact target amount.
________________________________________
Q2: Why use BFS?
A: BFS guarantees the minimum number of steps to reach the target — perfect for finding optimal solution.
________________________________________
Q3: What are the allowed operations?
A:
1.	Fill jug completely
2.	Empty jug completely
3.	Pour one jug into another until either source is empty or destination is full
________________________________________
Q4: Is every target amount achievable?
A: ✅ Only if target ≤ max(jug1, jug2) AND target is divisible by GCD(jug1, jug2).
Example: 4L & 3L → GCD=1 → any target 0-4 is possible.
________________________________________
Q5: Time and space complexity?
A:
•	States: (jug1+1) × (jug2+1) → O(jug1 × jug2)
•	BFS visits each state once → Efficient for reasonable jug sizes.
________________________________________
Q6: Can DFS be used?
A: Yes, but may find longer path or get stuck in cycles without visited tracking. BFS preferred for optimality.
________________________________________
Q7: What does (max(0, a - (jug2_cap - b)), min(jug2_cap, b + a)) mean?
A: It calculates result of pouring jug1→jug2:
•	jug2_cap - b = space left in jug2
•	If a > space, then a - space remains in jug1
•	Else, jug1 becomes 0, jug2 gets all a + b
